        the <package-name> from npm repository in our project and

2. what is a bundler ?
    - a bundler is a tool that collects all the files your web application needs, 
    processes them, and combines them into a small number of optimized files that 
    browser can efficiently load.

what does the bundler do ?
    - a bundler analyzes your project and builds a dependency graph.
    - a dependency graph is simply a map showing which files depend on which other files.
    - then a bundler > bundles all the fine into one seperate files
    instead of shipping 300 files, now the browser loads only a few optimized files.

examples of bundler > parcel, webpack, vite, etc.

what jobs does a bundler performs for us ?
    - combine files
    - resolve imports
    - minify code
    - remove unused code ( tree shaking )
    - optimize images
    - instead of loading many CSS files, a bundler combines them into one optimized css file.
    - they create Dev building
    - they host your application to the local server, on some project
    - auto refreshing you browser page > which is known as HMR ( hot module replacement )
        - this is happening becuase parcel is using file watching algorithm ( written in c++ )
    - reliable caching, see folder > .parcel-cache
    - consistent hashing
    - code splitting
    - differential bundeling > to support older browsers and different browsers
    - https support

- now you will see new problem > @parcel/transformer-js: Browser scripts cannot have 
imports or exports.
    - meaning > error means you're using ES module syntax (import/export) in a file 
    that Parcel is treating as a plain browser js script.
- solution :
    - there are two way we can run js on browser
    - first : classic old way > <script src="App.js"></script>
        Runs in global scope
        Cannot use import / export
    - second : ES Module modern way > <script type="module" src="App.js"></script>
        Runs in module scope
        Can use import / export
        Variables are isolated per file
- in our case Parcel sees your file has import/export statements, but your <script> tag 
is missing type="module" — so it throws this error because the two are incompatible.

why just .gitignore isn't enough ?
    - Think of Git like a watchlist. Once a file/folder is on the watchlist, 
    Git keeps monitoring it forever — even if you later tell it to ignore that file.
    - .gitignore only prevents new untracked files from being added. It has no effect 
    on files Git is already watching.

    - so you need to run command
    $ git rm -r --cached <file / folder_name>
    - meaning >
        - git rm : tells Git to remove something
        - -r : do it recursively ( for folders and all files inside )
        - --cached : remove from Git's tracking only, don't delete the actual folder
